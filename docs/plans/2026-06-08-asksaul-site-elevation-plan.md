## 0. Mission

Elevate AskSaul.ai from a competent AI/web/automation service site into a premium, practical, conversion-focused acquisition system.

The site should feel sharper, more valuable, and more trusted without losing AskSaul's strongest advantage: it is founder-led, operator-practical, and close to real business workflows.

The upgraded site must:

1. Convert more visitors into qualified leads.
2. Reframe the primary front door from a quote/proposal request into a valuable diagnostic.
3. Improve the homepage narrative and design hierarchy.
4. Add a named AskSaul method.
5. Improve accessibility, contrast, mobile navigation, keyboard support, and reduced-motion handling.
6. Improve SEO, structured data, sitemap coverage, and LLM-readability files.
7. Add a high-converting dumpster/waste voice-agent vertical page at `/voice-agents/waste`.
8. Preserve accurate claims and avoid generic AI hype.
9. Pass build/lint/accessibility/crawlability checks before handoff.

## 3. Named Method: The Saul Automation Map

AskSaul needs a proprietary-feeling but practical method.

### Public name

**The Saul Automation Map**

### Short explanation

The Saul Automation Map is a practical diagnostic for your business workflows. AskSaul looks at how leads come in, how customers get answers, how follow-up happens, and where the team is still doing repeatable work manually. Then we recommend what to automate first.

### Four-step method

1. **Spot the Leaks**
   - Identify where time, leads, and customer experience are slipping.
   - Examples: missed calls, slow response, repetitive questions, manual CRM updates, stale follow-up.

2. **Rank the Opportunities**
   - Prioritize based on revenue impact, time saved, implementation difficulty, and speed to value.

3. **Build the First System**
   - Turn the highest-priority opportunity into a working assistant, automation, voice agent, website flow, app, or marketing workflow.

4. **Improve From Real Usage**
   - Measure how customers and staff actually use the system, then refine.

### Homepage section copy

Eyebrow: `The Saul Automation Map`  
Headline: **Start with the map before you build the machine.**  
Body: `Most businesses do not need a giant AI overhaul. They need to find the daily friction that slows the team down and lets good leads slip away. The Saul Automation Map shows where to start, what to build first, and what to avoid overbuilding.`

## 6. Accessibility Requirements

Accessibility is part of conversion quality. The site must be usable and readable.

### Contrast

Minimum targets:

- Normal text: WCAG AA `4.5:1`
- Large text: WCAG AA `3:1`
- UI/focus indicators: `3:1`
- Prefer body text closer to `7:1` on dark backgrounds.

Current palette to audit:

- `--color-obsidian: #0A0A0F`
- `--color-carbon: #12121A`
- `--color-graphite: #1A1A24`
- `--color-cloud: #F3F5FA`
- `--color-slate: #C5CAD8`
- `--color-dim: #9399AA`
- `--color-cyan: #00D4AA`
- `--color-ice: #4AC6E8`

Tasks:

- Audit `text-dim`, `text-slate`, `text-cloud/60`, `text-slate/70`, and opacity-heavy copy.
- Avoid small gray text for important info, legal consent, CTA microcopy, and footer/contact details.
- Ensure placeholder text is readable.
- Ensure disabled button states do not rely only on opacity.

### Navigation accessibility

Known issue to fix:

- `components/layout/MobileNav.tsx` appears to expose a closed mobile nav dialog to the accessibility tree.

Tasks:

- When closed, do not render the dialog OR apply `hidden`, `inert`, and `aria-hidden="true"`.
- Add `aria-expanded` and `aria-controls` to hamburger button.
- Use a stable dialog ID.
- Move focus into the menu on open.
- Trap focus while open.
- Close on Escape.
- Restore focus to trigger on close.
- Ensure services dropdown is keyboard-accessible and not hover-only.

### Focus and motion

Tasks:

- Add/verify global `:focus-visible` styles for links, buttons, form fields, custom cards, and dropdown triggers.
- Add reduced motion handling in `app/globals.css`.
- Ensure scroll animations do not hide content if IntersectionObserver or motion logic fails.
- Browser screenshot QA must confirm pixels are visible, not just present in DOM.

### Form accessibility

Targets:

- Contact form
- Proposal/Automation Map builder
- Voice-agent lead form
- New waste-agent form

Tasks:

- Ensure every field has a label.
- Associate errors with fields.
- Ensure required fields are programmatically identified.
- Keep SMS consent clear and readable.
- Provide success/failure messaging.
- Do not silently succeed on failed lead submission in production.

## 11. Subagent Operating Plan

Avi stays orchestrator, integrator, reviewer, and QA owner. Subagents can help where workstreams are isolated.

### Suggested subagents

1. **Copy/conversion subagent**
   - Draft homepage sections, Automation Map copy, FAQ copy.
   - No code edits unless explicitly assigned.

2. **SEO/accessibility subagent**
   - Audit metadata, schema, robots, LLM files, nav ARIA, contrast.
   - Return file-specific recommendations and test commands.

3. **Waste vertical subagent**
   - Research dumpster demo sites.
   - Draft `/voice-agents/waste` page structure/copy/schema.
   - Verify demo phone/site info.

4. **Implementation subagent(s)**
   - Only after Avi splits tasks into non-overlapping files.
   - Examples:
     - One agent builds SEO helper/schema files.
     - One agent builds waste page route/components.
     - One agent patches accessibility/nav.
   - Avoid two agents editing same files.

### Subagent rules

Subagents must not:

- Push to remote.
- Deploy.
- Touch credentials.
- Invent claims, metrics, logos, testimonials, or pricing.
- Expose local OpenClaw gateway publicly.
- Rewrite broad site sections outside their task.
- Break existing form/API behavior without approval.

Avi must:

- Review every diff.
- Resolve conflicts.
- Run final quality gates.
- Browser-test the final integrated site.
- Report real command output.
